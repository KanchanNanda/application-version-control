Application Setup and Version Control Integration
Internship Capstone Project

This project demonstrates application development and complete version control management using Git and GitHub.

The main objective of this project is to develop a simple web application, manage source code using Git, implement a branching workflow, and maintain a structured version control process similar to real-world software development practices.

Project Objectives:-

- Develop a basic web application
- Initialize and manage a Git repository
- Push project code to GitHub
- Implement feature-based branching workflow
- Perform code changes using feature branches
- Merge feature branches into the main branch
- Maintain proper commit history and documentation

Git Branching Workflow:-
This project follows a feature-based branching workflow.
->Branches Used

Main Branch:
main
Contains stable and production-ready code.

Feature Branches:
feature/header
Used for adding website navigation and header components.

feature/contact
Used for implementing contact form functionality.


Git Workflow Implementation:-
The workflow followed:
------------------------------------------------------------------
Create Repository
|
↓
Develop Application
|
↓
Initialize Git Repository
|
↓
Create Feature Branch
|
↓
Make Changes
|
↓
Commit Changes
|
↓
Push Branch to GitHub
|
↓
Merge Feature Branch into Main
|
↓
Maintain Version History
-------------------------------------------------------------------

Git Commands Used:-

->Initialize Repository

git init
Check Repository Status
git status
Add Files
git add .
Commit Changes
git commit -m "commit message"
Create Branch
git checkout -b branch-name
Switch Branch
git checkout branch-name
Merge Branch
git merge branch-name
Push Code
git push origin branch-name
View Commit History
git log --oneline --graph --all
